What It Does

👤
Face Recognition
128-dimensional dlib embeddings identify registered subjects in real time. Falls back to OpenCV Haar cascades if dlib is unavailable.
🔥
Anomaly Detection
Custom YOLOv11 model detects fire, smoke, knives, and guns with configurable confidence thresholds.
📷
Live Camera Feed
Face recognition and anomaly detection run in parallel on every frame, with real-time identity and threat panels.
🗂
Identity Registry
Register, browse, and delete subjects entirely through the UI. Encodings stored locally with mtime-based cache.
🖼
Static Image Scan
Upload any image for combined face identification and anomaly scanning — no webcam required.
⚡
Smart Caching
The face database is cached in memory and reloads from disk only when the file changes — zero disk reads per frame.

How It Works

01 //
Capture
Frames are captured from a live webcam or a static image upload. Each frame is converted to RGB for face detection and kept as BGR for YOLO.
OpenCVStreamlit
02 //
Face Recognition
dlib's HOG detector locates faces, then encodes each into a 128-d vector. Euclidean distance to the identity registry determines identity — with OpenCV cosine-similarity as the no-dlib fallback.
dlibface_recognitionnumpy
03 //
Anomaly Detection
YOLOv11 runs inference on the same frame independently, returning bounding boxes and class labels for fire, smoke, knives, and guns above the confidence threshold.
YOLOv11UltralyticsPyTorch
04 //
Display & Alert
Results are merged onto the frame and streamed to the Streamlit dashboard. Identity and threat status cards update in real time with colour-coded severity.
StreamlitOpenCV

Get Running

# Clone and set up
$ git clone https://github.com/Ta-KC-trl/Black-eyes.git
$ cd Black-eyes
$ python -m venv .venv && .venv\Scripts\activate

# Install dependencies
$ pip install -r requirements.txt

# Launch dashboard
$ streamlit run Tracking.py
